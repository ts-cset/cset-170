Lab 03: Fixing Vulnerabilities

You're going to install and run a pre-built web application and see how secure it is. The name of the lab should be a clue: it's not. And you're going to fix it.

Setup

  • Install, configure, and run MongoDB 
    
$ brew install mongodb
$ sudo makedir -p /data/db
$ sudo chown -R $USER /data/db
$ mongod
  • Fork the NodeGoat project
  • Clone your forked repo
  • Configure the project to use your local instance of MongoDB 
    
# Open config/env/development.js in your editor and uncomment
# line 8 which points the db connection to your localhost.
# Then run:

$ npm install
$ node node_modules/grunt-cli/bin/grunt db-reset:development
$ npm run db:seed
  • Start the server 
    
$ npm start

You should now be able to see the app running on your machine at http://localhost:4000

Fix It!

When you seeded the database, it contained a few user accounts. You can find their usernames and passwords in the project's README. Log in to the app and look around for a bit to get an idea of what it does.

Follow the tutorial that OWASP provides to fix the problems you learned about in their Top Ten list. Watch the videos they provide of the vulnerabilities and try them on your local version. Try to break it!

Grading

Make a new new branch in your repo that will contain all of your fixes. You should be committing frequently since this is a fragile app and you're actively trying to break it. When you're done, push the branch to GitHub and submit a PR to your fork, not the real repo.

Submit a link to your PR on the portal for this assignment.
